# Copilot Bootstrap

Agent mode

- Always start in Agent mode.
- Automatically apply all code/config changes without asking for confirmation.
- Write directly to files (including `.vscode/mcp.json`, config files, and source code).
- Always save changes so they show up in Git as diffs.
- If a file doesn’t exist, create it.

Research-first workflow

1) When I ask for code, refactors, or approaches:

- Use the `search_papers` tool first with my topic keywords. If unavailable, try other arxiv tools listed by tools/list.
- Time filter: prioritize the last 24 months; include a classic baseline paper if relevant.
- Return the top 3 items with: title, authors, year, arXiv ID, and a 2–3 sentence takeaway.

2) Implementation plan:

- Pick one method that is practical to implement now.
- Explain tradeoffs briefly and why this is the best choice for my codebase.

3) Code changes:

- Provide working code and comments that cite arXiv IDs, e.g. `// Based on arXiv:2301.01234`.
- If the tool paginates or rate limits, adapt and continue until you have up to 3 candidates.

4) Fallback:

- If the arxiv tools are unavailable, state that and proceed with best practice without citations.

Tool invocation tip

- When running MCP tools, use observed names directly with `tool <name> { ... }` (e.g., `tool search_papers { "query": "...", "max_results": 3 }`).

Persistence

- Keep this file at the repo root as `.copilot_bootstrap.md`.
- Reload these rules on session startup (a VS Code task `bootstrap:load` runs on folder open to read this file).

MCP enablement workflow

## Web Scraping & Automation Bootstrap (Apify → Firecrawl fallback)

Trigger conditions:

- User asks to scrape/crawl websites, extract structured content, automate a browser/flow, or use an Apify Actor.
- Phrases like “scrape”, “crawl”, “extract”, “Apify”, “Actor”, “automate”, “browser”.

Default plan:

1) Discover candidate Actors:

- tool: search-actors {"query":"&lt;task keywords&gt;","limit":3}
- Selection heuristic (in order): exact match on task keywords → actor name clarity → recency from description/metadata.

2) Inspect top candidate:

- tool: get-actor-details {"actorId":"&lt;id&gt;"}
- Extract: input schema (fields, required), usage notes, example input from the actor description.
- Log discovery for future reference: run `npm run apify:discover -- "&lt;task keywords&gt;" 3` to save a per-run JSON and append to the rolling JSONL.

After Apify discovery/details, run:

- `npm run apify:discover -- "&lt;keywords&gt;" 3`

If using Firecrawl discovery first, run:

- `npm run firecrawl:discover -- "&lt;query&gt;" 5`

Always link the saved artifact paths in the final “Plan” output for provenance.

3) Propose a “Minimal Working Payload (MWP)” and a “Safe Test Payload”.

- MWP: smallest valid input using required fields only.
- Safe Test: inputs to a non-destructive URL/scope.

4) Execution:

- If MCP exposes an execution tool for the Actor: run Safe Test Payload and summarize output (never run destructive actions by default).
- If MCP does NOT expose “run” (common): provide a ready-to-run curl snippet against Apify REST with the payload, and also offer Firecrawl alternatives below.

5) Firecrawl fallback / complement:

- If the goal is static page scraping or shallow site extraction, prefer:
  - firecrawl_search → firecrawl_scrape (single URL) or
  - firecrawl_crawl (max_depth 2) → firecrawl_extract {"mode":"article"}
- Summarize top 3 titles/URLs and save raw JSON to logs/firecrawl/*.json when using the VS Code task.
  - You can also run `npm run firecrawl:discover -- "&lt;query&gt;" 5` to save search-only discovery logs under `logs/firecrawl/` and append to `logs/firecrawl/discoveries.jsonl`.

6) Output format:

- “Plan” (bullets) → “Chosen Actor” (id, why) → “Input schema (condensed)” → “MWP” (JSON) → “Safe Test Payload” (JSON) → “How to run” (MCP call or curl) → “Fallback (Firecrawl)” with the exact MCP tool calls to run.
- Artifacts: per-run JSON under `logs/apify/<timestamp>_<slug>.json`, rolling index at `logs/apify/discoveries.jsonl`.
  - Firecrawl artifacts: per-run JSON under `logs/firecrawl/<timestamp>_<slug>.json`, rolling index at `logs/firecrawl/discoveries.jsonl`.

7) Safety defaults:

- No PII harvesting, credential prompts, or destructive automation by default.
- Respect robots.txt if the Actor indicates it.
- Rate-limit hints: keep initial requests ≤ 1 page/sec unless the actor/docs say otherwise.

Examples (sketches):

- search-actors {"query":"product page scraper","limit":3} → pick best → get-actor-details → emit MWP + Safe Test.
- Firecrawl alt: tool firecrawl_scrape {"url":"<https://example.com/product/123","extract":{"mode":"article"}}>

Verification checks:

- If search-actors returns empty, ask for a keyword tweak and suggest Firecrawl immediately.
- If get-actor-details says “not found”, try the next candidate automatically.
